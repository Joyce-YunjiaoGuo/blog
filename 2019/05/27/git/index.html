git

发表于 2019-05-27 | 更新于 2019-06-04

上班也三周啦，上周结束了自己学习react-native的过程，开始接手项目。主要写了一个详情展示页面，没有什么交互，主要是结构和样式。哦还写了一个登录页的样式。

这过程中，最令我头大的，就是git命令不熟悉，连上传代码都需要带我的同事帮忙，十分挫败，于是，躺完了两天，周日的晚上总结一番。

git init

git clone

git branch newbranch 创建一个新的本地分支newbranch

git checkout newbranch 切换到newbranch分支，这两步可以合并为git checkout -b newbranch

在newbranch分支下，开发。开发完成后：

git add . 更新到cache
git commit -m “first-commit” 提交代码 -m==message 附言？

此时代码已提交到newbranch分支上，需要merge到远程分支xxx

git checkout master
git merge newbranch 把newbranch上代码合并到本地master分支
git push origin xxx 推代码到远程分支xxx

先切换到本地master， 合并newbranch上的代码到master，再将master push到远程的xxx分支

大功告成！以上是首次开发拉代码，推代码的过程。再次开发的话，可以采用如下拉取代码

git pull origin master 把远端origin的master分支拖动到本地当前分支

git branch test-0.40.0
git checkout test-0.40.0
git add .
git commit -m ‘test-0.40.0’
git push origin test-0.40.0
