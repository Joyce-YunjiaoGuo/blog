3.position： static/absolute/relative/fixed/inherit
absolute: 相对于最近的有定位的父级进行定位，如果没有，则相对于文档。脱离原位置。
relative: 相对于原来的位置（出生时）进行定位。保留原来位置。

4.css水平垂直居中
a.position定位 ie9以下不支持transform-translate 需要用margin-left/top宽高的一半

b.flex布局

5.margin塌陷： 父子嵌套，父垂直方向的margin，取两者最大值。当给子元素设置margin-top时，父子会一同向下移动，子相对于父不变。
解决方法： 给父级设置border或padding/BFC四选一。

margin合并： 兄弟元素垂直方向上的margin是合并的，取最大值。
解决方法： 添加父级BFC，但改变了html结构，不好/计算好间距即可。

6.浮动元素产生浮动流： 块级元素看不到，BFC和文本元素能看到
eg: box向左浮动，wrapper看不到，所有无法撑开。

清除浮动流： 给最后一个浮动元素设置伪元素

给父级设置BFC，父元素的宽高会紧紧抱住子元素。

7.块级元素： div/ul/li/h1-h6/p
行级元素： span/em/strong/a/br
行级块元素: img/input

9.inline元素设置width,height无效，垂直方向margin,padding无效， 水平方向margin,padding有效。

10.box-sizing: content-box; （标准盒模型）

11.BFC块级格式上下文
在父级上设： position: absolute;

12.px,rem,em的区别
px: 固定的像素，一旦设置就无法适应页面大小而改变
em/rem: 更灵活，适合响应式布局

em: 子元素字体大小的em相对于父元素字体大小
元素的width/height/padding/margin用em相对于该元素的font-size
rem: 相对于根元素

13.line-height继承的，最好不设单位，可随font-size比例缩放。

7.遍历数组 map vs forEach
map: 返回一个新数组（函数处理后的值），不检测空数组
forEach: 返回undefined，将每个元素传给回调，空数组不调用

8.for…in… vs for… of…
for…in…: 遍历对象，索引值为prop，会遍历原型上的属性和方法
for…of…: 遍历数组元素值， 不遍历原型上的属性和方法

9.let/const/var区别
var: 函数作用域，变量声明提升
let: 块级作用域，不存在变量声明提升（一定要先声明后使用），不可重复声明
const: 块级作用域，常量（声明必须赋值）

块级作用域： {…} 外面访问不到，报错ReferenceError

10.EventLoop：主线程从任务队列中读取事件，不断循环。
执行顺序： 一个macro-task => 所有micro-task => 一个macro-task …

macro-task: 主线程，setTimeout, setInterval, setImmediate, I/O, UI rendering
micro-task: process.nectTick, Promise, Object.observe, Mutation Observer
*promise的构造部分是同步执行的

17.TCP与UDP区别
TCP: 面向连接，正式收发数据前，必须建立可靠的连接（三次握手）。
UDP: 面向非连接，通信前无需建立连接，不管对方状态，直接发包。

18.常见状态码
2xx成功 – 200成功
204请求处理成功，但没有资源返回
206对资源某一部分请求
3xx重定向–301永久重定向，自动转到新位置
302临时重定向，继续用原有位置
304自从上次请求后，资源未修改，不返回网页内容
4xx客户端–400请求报文中存在语法错误
401需要http认证
403对请求资源的访问被服务器拒绝了
404页面不存在
5xx服务器–500内部资源故障
503服务器不可用（超负载/停机维护）

19.GET/POST区别
a.传参方式不同： GET使用url；POST直接放在请求主体中
b.POST比GET传输数据更多 因为GET用url传参，长度有限制；POST由于服务器安全性也有限制但比GET多
c.POST比GET安全 GET数据直接暴露在url上，可能CSRF攻击；POST数据在主体中较安全
d.POST比GET慢 POST在接受数据前，会把先将请求头发给服务器确认

21.从输入url到页面加载完成到底发生了什么？
1、浏览器的地址栏输入URL并按下回车。
2、浏览器查找当前URL是否存在缓存，并比较缓存是否过期。（expire/max-age last-modified/etag）
3、DNS解析URL对应的IP。
4、根据IP建立TCP连接（三次握手）。
5、HTTP发起请求。
6、服务器处理请求，浏览器接收HTTP响应。
7、渲染页面，构建DOM树。
8、关闭TCP连接（四次挥手）。
