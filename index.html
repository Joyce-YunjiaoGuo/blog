发表于 2019-10-26

最近公司的代码库，突然要求强制加hook，然后发现每次commit代码的message必须以指定代码开头，不然就无法push到远程。
一直报错： commit id cannot be empty。
但是！有一次填写commit message时，我加了-，就报错！于是我只能修改这一次的message，再重新push。
以前从来没这样操作过，于是，我去百度了一下：

git commit –amend 可以修改最近一次commit message

然而！进入了我不太会的vi界面！于是就有了这篇文章！

三种模式

基本上 vi/vim 共分为三种模式，分别是
命令模式（Command mode），
输入模式（Insert mode），
底线命令模式（Last line mode）。

这三种模式的作用分别是：

命令模式：

用户刚刚启动 vi/vim，便进入了命令模式。
此状态下敲击键盘动作会被Vim识别为命令，而非输入字符。

以下是常用的几个命令：

  • i 切换到输入模式，以输入字符。
  • x 删除当前光标所在处的字符。
  • : 切换到底线命令模式，以在最底一行输入命令。

命令模式只有一些最基本的命令，因此仍要依靠底线命令模式输入更多命令。

输入模式：

在命令模式下按下i就进入了输入模式。

在输入模式中，可以使用以下按键：

  • 字符按键以及Shift组合，输入字符
  • ENTER，回车键，换行
  • BACK SPACE，退格键，删除光标前一个字符
  • DEL，删除键，删除光标后一个字符
  • 方向键，在文本中移动光标
  • HOME/END，移动光标到行首/行尾
  • Page Up/Page Down，上/下翻页
  • Insert，切换光标为输入/替换模式，光标将变成竖线/下划线
  • ESC，退出输入模式，切换到命令模式

底线命令模式

在命令模式下按下:（英文冒号）就进入了底线命令模式。
在输入模式下按下shift + : 就进入了底线命令模式。

底线命令模式可以输入单个或多个字符的命令，可用的命令非常多。

在底线命令模式中，基本的命令有（已经省略了冒号）：

  • q 退出程序
  • w 保存文件
  • 按ESC键可随时退出底线命令模式

于是！想要修改最近一次commit message时，使用git commit –amend 进入vim后：

命令模式–按i进入输入模式–>
此时可以修改第一行的commit message–修改完成后，按shift+i进入底线命令模式–>
此时光标来到最后一行–输入wq!–>强制保存并退出

最近一次的commit message就修改完成啦~

这次操作只用了非常简单，为了以防万一，贴上vim比较全的操作用法。

1. 暴力解法：两层循环遍历 O(mn)

some() 方法会依次执行数组的每个元素：

注意： some() 不会对空数组进行检测。
注意： some() 不会改变原始数组。

重建二叉树

输入某二叉树的前序遍历和中序遍历的结果，请重建出该二叉树。假设输入的前序遍历和中序遍历的结果中都不含重复的数字。例如输入前序遍历序列{1,2,4,7,3,5,6,8}和中序遍历序列{4,7,2,1,5,3,8,6}，则重建二叉树并返回。

二叉树是每个结点最多有两个子树的树结构。通常子树被称作“左子树”（left subtree）和“右子树”（right subtree）。

所谓遍历二叉树，就是按一定的规则和顺序走遍二叉树的所有结点，使每一个结点都被访问一次，而且只被访问一次。

设L、D、R分别表示遍历左子树、访问根结点和遍历右子树， 则对一棵二叉树的遍历有三种情况：DLR（称为先根次序遍历），LDR（称为中根次序遍历），LRD （称为后根次序遍历）。

前：a-b-d-e-f-g-c
中：d-e-b-g-f-a-c
后：e-d-g-f-b-c-a

前序的第一个一定为root!
中序里root左边为左子树所有节点的中序遍历，右边为右子树所有节点的中序遍历。

思路：
首先从pre中选取第一个，pre[0]为root; 从mid中找出它的index, 由此把mid分为left,right两个子树；根据两个子树的长度，从pre中截出对应的子树；递归。

一开始，命名为Oak(因为看见了窗前的一棵橡树？)，但是已经被注册了，只能换名。1995年时，正式命名为Java，希望这种语言像爪哇的猫屎咖啡一样盛行于全球，这么看来很成功呀？（想起了UCI的小卖部java city）

特点

  1. 跨平台性（不同厂商的芯片–> 不同版本的操作系统）

java为什么能跨平台呢？

因为虚拟机JVM可以在内存上开辟一块空间，将 源文件（.java）编译成 字节码文件（.class），使其适用于不同的操作系统。

想要运行java程序，需要安装java运行环境jre(包含jvm)

想要做java开发， 需要安装java开发工具jdk(包含jre)

jdk中都有啥呢？

– javac.exe(执行工具)
– include(其他语言写好的程序)
– jre(运行环境)
– lib(别人写好的所有java类)
– src.zip(java源代码)
